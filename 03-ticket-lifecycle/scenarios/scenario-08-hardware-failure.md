# Scenario 8: Hardware Failure (Escalation Demo)

## Ticket Summary
- **Submitted by:** Eric Nguyen (IT Department)
- **Help Topic:** Equipment Request
- **SLA:** SEV-B (High)
- **Priority:** High
- **Assigned To:** Help Desk / Level I → Systems Administration / Level II (escalated)

## Problem Description
Eric Nguyen reports that his workstation randomly crashes with a blue screen error. The crashes have happened multiple times this week and are disrupting his work.

## Steps Taken to Resolve

### Step 1: Ticket Triage (Level I)
- Screenshot: Ticket as it appears in the agent queue
- Observations: Note the auto-assigned priority, SLA, and department
- Actions: Adjust priority/SLA/assignment if needed, document why

### Step 2: Initial Response (Level I)
- Screenshot: Internal note or reply to user
- Actions: Gather information — Event Viewer logs, crash dump details, frequency of crashes

### Step 3: Escalation to Level II
- Screenshot: Escalation note with findings attached
- Actions: Document findings in an internal note, escalate to Level II / Systems Administration with rationale

### Step 4: Level II Investigation
- Screenshot: Event Viewer analysis, hardware diagnostic results
- Actions: Review escalation notes, analyze Event Viewer logs, determine hardware issue, document recommendation (e.g., replace RAM, submit warranty claim). Update priority to High.

### Step 5: Resolution & Closure
- Screenshot: The closed ticket showing resolution notes from Level II
- Actions: Document the resolution summary, escalation path taken, and final recommendation
